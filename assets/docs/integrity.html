🔐 Integrity Plugin

Secure REST API bridge for the Unity plugin suite. Integrity provides authenticated, rate-limited access to Groups, Meetings, Members, Positions, and Intergroup Meetings for external applications.

What Integrity Does

Integrity exposes Unity's data through a standard REST API at /wp-json/integrity/v1/. Every request must carry a valid API key. Keys are scoped with granular permissions, protected by rate limiting, and optionally restricted to specific IP addresses. Every API call is recorded in an audit log.

The plugin also ships with a .NET client library for consuming the API from C# applications.

Requirements

  • WordPress 6.0 or later
  • PHP 8.0 or later (Argon2id password hashing required)
  • Unity plugin — must be installed and activated
  • Scrutiny plugin — listed as a required dependency
  • HTTPS — strongly recommended and enforced by default (see Settings)

Installation

  1. Make sure Unity and Scrutiny are installed and activated.
  2. Upload the integrity folder to /wp-content/plugins/, or install via the WordPress plugin uploader.
  3. Activate Integrity in the Plugins screen.
  4. On activation, three database tables are created: wp_integrity_api_keys, wp_integrity_rate_limits, and wp_integrity_audit_log. A daily cleanup cron job is also scheduled.
  5. A new Integrity API top-level menu appears in the admin sidebar with sub-pages for API Keys, Audit Log, and Settings.

Managing API Keys

Navigate to Integrity API → API Keys. This page lists all keys with their name, prefix, permissions, rate limit, last-used date, lifetime request count, expiration status, and active/revoked state.

API keys are generated with a cryptographically secure random value, prefixed with int_. Only the Argon2id hash is stored in the database — the plain-text key is shown once at creation and cannot be retrieved afterwards.

🚨
Copy the key immediately after creation. It is displayed only once. If you lose it, you must create a new key.

Creating a Key

On the API Keys page, fill in the creation form:

FieldDescription
NameA human-readable label for the key (e.g. "Mobile App", "Reporting Server").
PermissionsCheckboxes for each resource. See Permissions.
Rate LimitRequests per hour. Leave blank to use the global default (1,000).
Expires AtOptional expiration date. After this date, the key will fail validation.
IP WhitelistOptional. One IP or CIDR range per line. See IP Whitelisting.

Permissions

Each API key is scoped to specific resources and actions:

PermissionTypeGrants Access To
groups:readReadList and view groups
meetings:readReadList and view meetings
positions:readReadList and view positions
members:readReadList and view members
members:writeWriteUpdate member data
intergroup-meetings:readReadList and view intergroup meetings
intergroup-meetings:writeWriteRegister/unregister group and officer attendance
*AllFull access to all endpoints

If no permissions are selected when creating a key, it defaults to groups:read and meetings:read.

IP Whitelisting

Optionally restrict a key so it can only be used from specific IP addresses. Enter one IP or CIDR range per line when creating or editing a key:

203.0.113.50
10.0.0.0/8
2001:db8::/32

Both IPv4 and IPv6 are supported, including CIDR notation. If the whitelist is set and the client's IP doesn't match any entry, the request is rejected — even if the key is otherwise valid.

Revoke & Delete

Revoking a Key

Revoking sets the key to inactive. It can no longer authenticate requests, but the record is kept for audit purposes. You can see it was revoked, when it was last used, and how many requests it made.

Deleting a Key

Permanently removes the key record from the database. Use this for keys you no longer need to audit. This action cannot be undone.

Using the API

Pass your API key in the Authorization header using the Bearer scheme:

GET /wp-json/integrity/v1/groups
Authorization: Bearer int_a1b2c3d4...

Alternatively, you can use the X-API-Key header:

X-API-Key: int_a1b2c3d4...

All successful responses follow this shape:

{
  "success": true,
  "data": [ ... ],
  "meta": { "total": 42, "page": 1, "per_page": 100 }
}

Error responses include a machine-readable code and human-readable message:

{
  "success": false,
  "error": { "code": "invalid_api_key", "message": "Invalid or expired API key" }
}

Rate Limiting

Each API key has a per-hour request limit (default: 1,000). Integrity uses a sliding-window approach rounded to the current clock hour. Every response includes rate limit headers:

HeaderMeaning
X-RateLimit-LimitThe maximum requests allowed in the current window.
X-RateLimit-RemainingHow many requests are left in the current window.
X-RateLimit-ResetUnix timestamp when the current window resets.

When the limit is exceeded, the API returns HTTP 429 Too Many Requests. Wait until the reset timestamp before retrying.

Security Headers

All Integrity API responses include these security headers automatically:

  • X-Content-Type-Options: nosniff
  • X-Frame-Options: DENY
  • X-XSS-Protection: 1; mode=block
  • Cache-Control: no-store, no-cache, must-revalidate, max-age=0

This prevents browsers from caching API responses or embedding them in frames.

Health Check

GET /integrity/v1/health

No authentication required. Returns the API status, version, and whether Unity is available. Returns 200 when healthy, 503 when degraded (Unity unavailable).

Groups

Permission required: groups:read

GET /integrity/v1/groups

List all groups. Supports pagination, search, district filtering, and optional meeting expansion.

Parameters: per_page (1–500, default 100), page, search, district_id, expand=meetings

GET /integrity/v1/groups/{id}

Get a single group by ID. Supports expand=meetings to include full meeting objects instead of IDs.

Group responses include: id, title, email, phone, website, link, notes, district_id, last_contact, meeting IDs (or full meeting objects if expanded), contacts, and contribution options (Venmo, PayPal, Square).

Meetings

Permission required: meetings:read

GET /integrity/v1/meetings

List all meetings. Supports pagination, search, and filtering by day, group, and type.

GET /integrity/v1/meetings/{id}

Get a single meeting by ID.

Positions

Permission required: positions:read

GET /integrity/v1/positions

List all intergroup positions. Supports pagination and search.

GET /integrity/v1/positions/{id}

Get a single position by ID.

Members

Permission required: members:read for reading, members:write for updating.

GET /integrity/v1/members

List all members. Supports pagination, search, and filtering by home group, position, and GSR status.

GET /integrity/v1/members/{id}

Get a single member by ID.

POST /integrity/v1/members/{id}/update

Partial update — only send the fields you want to change. Omitted fields keep their current values. Referenced entities (home_group_id, intergroup_position_id) are validated to exist before saving. Obscured personal data values (from Scrutiny masking) are detected and ignored to prevent overwriting real data with bullet characters.

💡
The update endpoint is smart about Scrutiny's data obscuring. If the submitted personal_email or mobile_number contains bullet characters (•), the value is treated as obscured and the existing value is preserved.

Intergroup Meetings

Permission required: intergroup-meetings:read for reading, intergroup-meetings:write for registration.

GET /integrity/v1/intergroup-meetings

List intergroup meetings with attendance data. Supports pagination and date filtering.

GET /integrity/v1/intergroup-meetings/{id}

Get a single intergroup meeting with full attendance details.

POST /integrity/v1/intergroup-meetings/{id}/register-group

Register a group's attendance at an intergroup meeting.

POST /integrity/v1/intergroup-meetings/{id}/unregister-group

Remove a group's attendance registration.

POST /integrity/v1/intergroup-meetings/{id}/register-officer

Register an officer's attendance at an intergroup meeting.

POST /integrity/v1/intergroup-meetings/{id}/unregister-officer

Remove an officer's attendance registration.

.NET Client Library

Integrity ships with a typed C# client library (UnityRestSharp) and a set of model classes for consuming the REST API from .NET applications. The client handles authentication, JSON serialisation, rate limit header parsing, and error handling out of the box.

ComponentLocationDescription
Client Librarysharp/TheBleedingDeacons.Unity.Client/The UnityRestSharp class — a full typed API client.
Modelssharp/TheBleedingDeacons.Unity.Models/C# record/class definitions for Group, Meeting, Member, Position, IntergroupMeeting, etc.
💡
The client targets .NET 9 with C# 13 language features. It uses System.Text.Json for serialisation with snake_case property naming to match the WordPress API convention.

Setup

Add a project reference to both the Client and Models projects, or copy the source files into your solution:

// In your .csproj
<ItemGroup>
  <ProjectReference Include="..\..\sharp\TheBleedingDeacons.Unity.Client\TheBleedingDeacons.Unity.Client.csproj" />
  <ProjectReference Include="..\..\sharp\TheBleedingDeacons.Unity.Models\TheBleedingDeacons.Unity.Models.csproj" />
</ItemGroup>

Create a client instance with your site URL and API key:

using TheBleedingDeacons.Unity.Client;

using var client = new UnityRestSharp(
    "https://example.com",   // WordPress site URL
    "int_a1b2c3d4e5f6..."   // Your Integrity API key
);

The client sets the Authorization: Bearer header automatically, along with Accept: application/json and a IntegrityClient/1.0 user-agent string. It implements IDisposable so you can use it in a using block or inject it via DI.

Dependency Injection

You can pass an existing HttpClient instance for DI scenarios (e.g. IHttpClientFactory):

var client = new UnityRestSharp("https://example.com", apiKey, myHttpClient);

Usage Examples

Health Check

var health = await client.CheckHealthAsync();
Console.WriteLine($"Status: {health?.Status}, Unity: {health?.UnityAvailable}");

List Groups (with expanded meetings)

var response = await client.GetGroupsAsync(expandMeetings: true);

if (response.Success && response.Data != null)
{
    foreach (var group in response.Data)
    {
        Console.WriteLine($"{group.Title} — {group.Meetings.Count} meetings");
    }
}

Filter Meetings by Day and Online Status

var response = await client.GetMeetingsAsync(
    dayOfWeek: DayOfWeek.Monday,
    online: true,
    perPage: 50
);

Get a Single Member

var response = await client.GetMemberAsync(id: 42, expandHomeGroup: true);

if (response.Success && response.Data != null)
{
    var member = response.Data;
    Console.WriteLine($"{member.AnonymousName} — {member.HomeGroupName}");
}

Update a Member (partial update)

var response = await client.UpdateMemberAsync(42, new UpdateMemberRequest
{
    AnonymousName = "John D.",
    IsGsr = true,
    HomeGroupId = 15
});

Register Group Attendance at an Intergroup Meeting

var response = await client.RegisterAttendeeAsync(
    intergroupMeetingId: 7,
    memberId: 42,
    meetingGroup: "Tuesday Night Group",
    gsrName: "John D.",
    gsrProxy: false
);

Register an Officer

var response = await client.RegisterOfficerAsync(
    intergroupMeetingId: 7,
    officerId: 99,
    positionName: "Treasurer",
    officerName: "Jane S."
);

Response Handling

Every method returns an ApiResponse<T> wrapper that contains:

PropertyTypeDescription
SuccessboolWhether the request succeeded (HTTP 2xx).
DataT?The typed response data (null on failure).
ErrorApiError?Error details with Code and Message (null on success).
MetaResponseMeta?Pagination metadata: Total, Page, PerPage, TotalPages.
StatusCodeintHTTP status code (0 for network/parse errors).
RateLimitRateLimitInfo?Parsed rate limit headers: Limit, Remaining, Reset, and ResetDateTime.

Error Handling Pattern

var response = await client.GetGroupsAsync();

if (!response.Success)
{
    Console.WriteLine($"Error {response.StatusCode}: {response.Error?.Code} — {response.Error?.Message}");

    // Check rate limiting
    if (response.StatusCode == 429)
    {
        Console.WriteLine($"Rate limited. Retry after {response.RateLimit?.ResetDateTime}");
    }
    return;
}

// Process data
foreach (var group in response.Data!)
{
    Console.WriteLine(group.Title);
}

The client catches both HttpRequestException (network errors) and JsonException (parse errors) internally and wraps them in the ApiResponse with appropriate error codes (network_error / parse_error), so you never need to wrap calls in try-catch.

Audit Log

Navigate to Integrity API → Audit Log. Every API request is recorded with:

ColumnDescription
Date / TimeWhen the request was made.
API KeyWhich key was used (by ID), or blank for failed authentication.
EndpointThe REST route that was requested.
MethodHTTP method (GET, POST).
Response CodeHTTP status code returned (200, 401, 403, 404, 429, 500, etc.).
Response TimeHow long the request took to process (in seconds).
IP AddressThe client's IP address (supports proxy headers like Cloudflare).
User AgentThe client's user-agent string (truncated to 500 characters).
Request ParamsSanitised request parameters. Sensitive keys (password, token, key, etc.) are redacted.

The audit log page includes filters for API key, response code, IP address, and date range. A 30-day statistics summary is shown at the top with total requests, success rate, failed auth attempts, rate-limited requests, average response time, and top endpoints/IPs.

You can clear logs by age (older than N days) or by API key from the audit log page.

Settings

Navigate to Integrity API → Settings. Four configurable options:

SettingDefaultDescription
Require HTTPSOnReject API requests over plain HTTP. Bypassed when WP_DEBUG is true (for local development).
Default Rate Limit1,000Requests per hour for newly created keys. Can be overridden per key.
Enable Audit LogOnLog every API request. Disable to save database writes on very high-traffic sites.
Log Retention90 daysLogs older than this are automatically deleted by the daily cron job.

The settings page also shows the REST API base URL, plugin version, and Unity plugin status for quick diagnostics.

Automatic Cleanup

A daily WordPress cron job runs automatically and performs two tasks:

  • Rate limit records — entries older than 1 day are deleted from the wp_integrity_rate_limits table.
  • Audit log entries — entries older than the configured retention period (default 90 days) are deleted from wp_integrity_audit_log.

The cron is scheduled on plugin activation and cleared on deactivation.

Troubleshooting

401 "Invalid or expired API key"

Check that you're sending the full key (including the int_ prefix) in the Authorization header. If the key has an expiration date, verify it hasn't passed. If the key was revoked, it will also return 401.

403 "HTTPS is required"

The "Require HTTPS" setting is on and you're making a request over plain HTTP. Either use HTTPS or disable the setting (not recommended for production). During development with WP_DEBUG enabled, this check is bypassed.

403 "Insufficient permissions"

Your API key doesn't have the required permission for this endpoint. Go to Integrity API → API Keys and check the permissions assigned to this key.

429 "Rate limit exceeded"

You've exceeded the hourly request limit for this key. Check the X-RateLimit-Reset header for when the window resets. You can increase the per-key rate limit on the API Keys page.

API returns 503 on health check

Unity is not available. Verify that the Unity plugin is activated. The health endpoint will return "status": "degraded" with "unity_available": false.

IP whitelist blocking legitimate requests

If your client is behind a proxy or load balancer, the IP seen by Integrity may differ from your expected IP. Integrity checks Cloudflare (CF-Connecting-IP), X-Forwarded-For, and X-Real-IP headers in that order. Verify which IP is being logged in the audit trail and add it to the whitelist.

FAQ

Can I retrieve a key after creation?

No. Only the Argon2id hash is stored. If you lose the key, revoke the old one and create a new one.

Does the audit log store personal data?

Request parameters are sanitised before logging — keys containing words like "password", "token", "secret", or "credit_card" are automatically redacted. Long values are truncated to 200 characters.

What happens if I send an obscured email/phone to the update member endpoint?

Integrity detects values containing bullet characters (•) — the masking pattern used by Scrutiny — and preserves the existing value instead of overwriting it. This prevents accidentally replacing real data with masked placeholders.

Can I have multiple API keys?

Yes. Create as many keys as you need — one per application or integration is a good practice. Each key has its own permissions, rate limit, IP whitelist, and expiration.

What's the expand parameter on groups?

By default, group responses include just meeting IDs. Adding ?expand=meetings replaces those IDs with full meeting objects, saving you from making separate requests for each meeting.

Where's the .NET client library?

See the .NET Client Library section above. The client library is in sharp/TheBleedingDeacons.Unity.Client/ and the models in sharp/TheBleedingDeacons.Unity.Models/.
